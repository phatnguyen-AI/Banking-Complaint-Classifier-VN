**Project Name:** Banking Complaint Classifier for Vietnamese Text

**Author:** Phat Nguyen | tanphat6406@gmail.com | LinkedIn: https://www.linkedin.com/in/phat-nguyen-a264722b7/

**Business Problem:**  
- Vietnamese banks receive thousands of customer complaints daily across channels (app, email, hotline, branch).  
- Manual routing of complaints to the correct department (Card, Fraud, Loan, etc.) is slow, error-prone, and expensive.  
- Misrouted complaints increase resolution time, reduce customer satisfaction (NPS), and create compliance risk.

**Solution:**  
- An end-to-end NLP system that automatically classifies Vietnamese banking complaints into 6 categories and routes them to the appropriate department in real time.  
- 6 categories: CARD_ISSUE, APP_LOGIN, TRANSACTION, LOAN_SAVING, FRAUD_REPORT, OTHERS.

**Technical Stack & Decisions (explain the WHY behind each choice):**  
1. Base Model: `paraphrase-multilingual-MiniLM-L12-v2` — chosen over PhoBERT/XLM-R because it offers multilingual Vietnamese support with a much smaller footprint, critical for cost-constrained deployment.  
2. Fine-tuning: Transfer learning on ~4,640 Vietnamese banking complaint samples. Train/Val/Test split: 80/10/10. Hyperparams: lr=2e-5, batch=32, epochs=5, max_len=64.  
3. ONNX + INT8 Dynamic Quantization: Reduced model size by ~75% while maintaining ~87% accuracy (F1 ~87%). Enables inference <100ms per request.  
4. ONNX Runtime with single-thread config (`OMP_NUM_THREADS=1`, `intra_op_num_threads=1`, `enable_cpu_mem_arena=False`) — optimized for 0.1 CPU / 512MB RAM free-tier hosting.  
5. Backend: FastAPI with async lifecycle management, CORS middleware, softmax post-processing.  
6. Frontend: Vanilla HTML/CSS/JS + Bootstrap 5 on Vercel CDN. Responsive, real-time API health check (30s polling), graceful error handling.  
7. Deployment: Backend on Render (Free Plan, Docker container, auto-deploy on push), Frontend on Vercel (auto-deploy). Zero hosting cost.  
8. Training Pipeline: Modular Python pipeline — `cleaning/` → `training/` → `quant/` → production-ready ONNX model. Fully reproducible.

**Performance Metrics:**  
- Accuracy: ~87%  
- F1-score: ~87%  
- Inference latency: <100ms per request  
- Model size after quantization: ~129MB  
- Memory footprint: <100MB per inference  
- Runs stable on 0.1 CPU, 512MB RAM (Render Free Tier)
